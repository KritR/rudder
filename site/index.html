Native worker panes for real agent CLIs

The terminal OS for coding agents.

Rudder runs Claude Code and Codex side by side in actual terminal panes, gives every task an isolated git worktree, and keeps review, merge, and cloud handoff in one focused dashboard.

npm install -g @viraatdas/rudder@latest rudder

Option-1/2/3 Direct focus for agents, worker, and task panes.
Tab in worker Forwarded to Claude Code or Codex, so native queues and menus keep working.
Wheel / trackpad Scrolls the pane under the pointer without stealing focus.
/main or /m Starts a main-branch agent and groups it separately from worktrees.

Built around real terminal sessions.

Rudder does not replay a simplified transcript UI. The worker pane is the harness itself, with Rudder handling the outer workflow: pane focus, scrollback, worktree isolation, task summaries, review, merge, and cloud continuation.

parallel

Task isolation

Each normal task gets a git worktree, run metadata, and shared RUDDER.md context so agents can work concurrently without trampling the main checkout.

native

Claude and Codex

Claude Code and Codex stay native. Their prompts, slash commands, cursor movement, selection, and Tab behavior remain inside the worker pane.

review

Diff and merge

Toggle live review, inspect changes, merge one completed worktree, or combine all completed worktrees when the run is ready.

The controls stay predictable.

Focus changes are explicit. Mouse hover does not move focus, and trackpad scrolling feels like terminal pane scrollback instead of child-app roulette.

Focus

Option-1/2/3Focus agents, worker, or task directly.
Tab / Shift+TabForwarded to the focused worker TUI.
Ctrl-GToggle Rudder nav mode while inside a worker.
Mouse hoverNever changes pane focus.
Ctrl-CLeave Rudder from any pane.

Agents

j / kSelect an agent in the left pane.
vOpen or close live diff review.
m / MMerge selected worktree, or all completed worktrees.
RCombine completed worktrees and start a review-all agent.
ddDelete an agent and remove its worktree after confirmation.

Task

/modelChoose provider, model, and effort.
/main or /mSpawn a main-branch agent using the selected model.
/planUse native read-only planning before implementation.
/cloudOnload this workspace, or start fresh on Fly.
/usageShow tokens and estimated cost for this session.

Scroll

wheelScroll the pane under the pointer.
trackpadMoves Rudder pane history one terminal row at a time.
PageUp/PageDownScroll the focused worker by roughly one page.
selection dragCopies text from worker scrollback.

Worktrees today, JJ on the roadmap.

Rudder's current concurrency model is intentionally git-native. Issue #7 tracks adding a JJ workspace mode so teams that live in jj can keep the same Rudder dashboard while using the jj CLI as the workspace interface.

Shipping path

Normal agents run in isolated git worktrees. Main agents created with /main or /m stay grouped in the agents pane so base-branch conversations are separate from task worktrees.

Tracked next

Native JJ workspace support is open and tracked in issue #7. The goal is to let Rudder multiplex work through jj without turning the worker pane into a separate implementation.
